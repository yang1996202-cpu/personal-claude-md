# personal-claude-md

我的 Claude Code 个人工作契约——主文件 `CLAUDE.md` + 5 个子文档，约束 Agent 在我这台机器上的工作方式。

> 当参考看思路，别原样照搬。里面有我的本机路径、工具链和个人偏好，照搬前请按你自己的方式改。

## 解决什么

Agent 写得快但容易：上下文一长就重复造轮子、改东西不告知、凭记忆断言、目标没对齐就动手。这套契约用工程化约束治这些。

核心思路：**主文件只放每次必加载的硬规则 + 路由，深度细则下沉子文档**，避免主文件臃肿、子文档重复。

## 结构

```
CLAUDE.md              主文件：硬规则 + 子文档路由（按 基础→控场→纪律→辅助→收束 分组）
code-discipline.md     代码纪律（先讲假设 / 精准手术 / 查重 / 调试查链路 / 代码审查）
tool-usage.md          工具使用（诊断流程 / context7 / MCP / 联网抓取能力链）
writing-and-lang.md    长文与语言（写作风格；对话风格归主文件第 2 节）
memory-and-docs.md     记忆与文档（gbrain 存取 / 身份文件改动）
mode-routing.md        模式路由（攻坚 / 定案 + 对齐刹车）
```

## 几个关键设计

- **可见性硬规则**：加载子文档、派子代理前必须单行声明原因（`[加载:…]` / `[路由:…]`），不准默默触发。
- **攻坚 / 定案 两模式**：去掉「探索」模式（模型默认就探索），攻坚承载日常摸索、定案要求完整交付（禁止半成品 / 禁止分期）。
- **对齐刹车**：目标不清时先「提问 → 复述 → 等许可」，再动手。
- **子文档路由**：主文件不堆细则，按任务类型指向对应子文档（文件名 — 定位 — 何时读）。
- **去重**：主文件工具 / 编码节瘦身为「指针 + 硬规则」，细则全在子文档，不两边重复。
